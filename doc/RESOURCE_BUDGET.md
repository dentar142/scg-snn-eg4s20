# EG4S20BG256 资源预算

## 1. 器件参数（来自 Anlogic EG4 系列 datasheet）

| 资源 | 数量 |
|---|---|
| LUT4 | **19,712** |
| 寄存器 (DFF) | ~14,000 |
| Block SRAM (B9K, 9 Kbit) | **64** 块 ≈ 576 Kbit ≈ 72 KB *（注：早期资料给 ~100 Kbit，本文按 EG4S20 规范取 64 块 B9K 上限。请以 TD 6.2.2 报告为准）* |
| 18×18 乘法器 (DSP18) | **9** |
| PLL | 4 |
| GPIO（BG256 封装）| ~178 用户 I/O |
| 工艺 | 55 nm 类 |

> 注：EG4S20 的 BRAM 由若干 B9K（9 Kbit）和 B16K（16 Kbit）混合组成；**实际可用值以 TD 综合报告为准**。本文按保守估计 ~64 KB 可用做规划。

## 2. 模型参数预算

模型架构（v0，刻意保守以保证一次综合通过）：

| 层 | 类型 | 输入 | 输出 | 卷积核 | 参数（INT8） |
|---|---|---|---|---|---|
| L0 | Conv1D | 1 × 256 | 8 × 256 | 5 | 1×8×5 = 40 B |
| L1 | Conv1D | 8 × 128 | 16 × 128 | 5 | 8×16×5 = 640 B |
| L2 | Conv1D | 16 × 64 | 16 × 64 | 5 | 16×16×5 = 1,280 B |
| L3 | Conv1D (1×1) | 16 × 32 | 3 × 32 | 1 | 16×3×1 = 48 B |
| GAP+FC | — | 3 × 32 | 3 logits | — | 3×3 = 9 B |
| **总计权重** | | | | | **~2 KB** |
| **偏置（INT16）** | | | | | 8+16+16+3+3 = 46 × 2 = ~100 B |
| **缩放因子（INT16）** | | | | | 4 layers × 2 B = 8 B |
| **总片上常量** | | | | | **~2.2 KB** |

中间激活内存（Ping-Pong 双 buffer）：
- 最大单层激活：8 × 256 × 1 B = 2,048 B
- Ping-Pong：2 × 2,048 = **4,096 B**

**片上总存储需求 ≈ 2.2 KB（权重）+ 4 KB（激活）= 约 6.2 KB**，远小于 EG4S20 的 64 KB 余量——可全片上、不依赖外部 PSRAM。

## 3. 计算资源预算

### 3.1 MAC 阵列规模

- 4 个 PE × 1 MAC/cycle = **4 MAC/cycle**；
- 每个 PE 用 1 个 EG_PHY_MULT（18×18 乘法器） + 累加 DFF 实现；
- **总 DSP 占用：4 / 9**，留 5 个余量给后续扩展（softmax / softmax 替代 / BatchNorm fold）。

### 3.2 时延估算

| 层 | 输入长度 | 输入通道 | 输出通道 | 卷积核 | MACs | 周期数 (4 PE) |
|---|---|---|---|---|---|---|
| L0 | 256 | 1 | 8 | 5 | 256 × 8 × 5 = 10,240 | ~2,560 |
| L1 | 128 | 8 | 16 | 5 | 128 × 16 × 8 × 5 = 81,920 | ~20,480 |
| L2 | 64 | 16 | 16 | 5 | 64 × 16 × 16 × 5 = 81,920 | ~20,480 |
| L3 | 32 | 16 | 3 | 1 | 32 × 3 × 16 = 1,536 | ~384 |
| **总计** | | | | | **~175 K** | **~44 K** |

加 priming（pipeline 填充）+ requantization 开销，单次推理约 **50–60 K 周期**。

@ 50 MHz 时钟：**~1.0 – 1.2 ms / 推理**——比论文（95.5 ms @ 24 MHz）**快约 80×**——这是因为模型参数缩了一个量级。

### 3.3 LUT4 占用估算

| 模块 | 估算 LUT4 |
|---|---|
| 4-PE MAC 阵列（控制 + 累加） | ~600 |
| Requantization & 激活（ReLU + shift） | ~400 |
| 控制 FSM + 层间序列器 | ~300 |
| Ping-Pong 缓冲控制 | ~200 |
| UART RX/TX | ~250 |
| 顶层 wire / glue | ~300 |
| **总计** | **~2,050 LUT4 ≈ 10% 占用** |

资源利用率非常宽松——v0 一次综合通过的把握很高。

## 4. 时钟与功耗预估

- 内部时钟：用 `EG_PHY_PLL` 把 24 MHz 板载晶振倍频到 **50 MHz**；
- UART 时钟：单独由 `EG_PHY_PLL` 输出到 **115.2 kHz × 16 = 1.8432 MHz** 用于 UART 过采样；
- 静态功耗（55 nm）：≈ 30 mW；
- 动态功耗（50 MHz、10% 动态率）：≈ 50–80 mW；
- **预估总功耗 ≈ 80–120 mW**（远高于 iCE40UP5K 的 8.55 mW，但仍属嵌入式可接受范围）。

## 5. 与原论文的差距与本工程的"取舍"

| 维度 | 论文（iCE40UP5K） | 本工程（EG4S20） | 差距来源 |
|---|---|---|---|
| 验证准确率 | 98% | 目标 ≥ 90% | 模型缩了一个量级 |
| 推理时延 | 95.5 ms @ 24 MHz | ~1.2 ms @ 50 MHz | 模型小 + 时钟高 |
| 功耗 | 8.55 mW | ~80–120 mW | 工艺差距（40 nm vs 55 nm + 非 ULP 设计） |
| LUT 占用 | 2,861 / 5,280 (54%) | ~2,050 / 19,712 (10%) | EG4S20 资源更多 |
| DSP 占用 | 7 / 8 (87%) | 4 / 9 (44%) | 模型小 |
| 是否需要外部存储 | 否 | 否 | 全片上 |

**结论**：在 EG4S20 上做"类似实现"完全可行，且综合压力远低于原论文在 iCE40 上的极限挑战。
